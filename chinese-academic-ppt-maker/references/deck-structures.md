# Deck Structures

## Project Handoff / Reproduction Progress

Use 8-10 slides:

1. `我接手后先做了三件事`
2. `项目问题：平均有效不等于人人有效`
3. `数据来自哪项研究，结局如何定义`
4. `建模流程：从基线变量到治疗效应分层`
5. `训练/测试如何划分`
6. `复现到哪一步，哪些结果已对齐`
7. `关键图表逐一解释`
8. `当前不能过度表述的边界`
9. `待确认问题`
10. `下一步计划`

Use this when the user has just taken over a project or needs a group meeting update.

## Learning Report

Use 6-8 slides:

1. `我原来不清楚的问题`
2. `核心概念一：...`
3. `核心概念二：...`
4. `这个概念在项目中如何落地`
5. `我现在能解释的图和数`
6. `还没完全搞清楚的部分`
7. `下一步怎么学/怎么复现`

Keep humility and learning trajectory visible.

## Manuscript Pitch

Use 10-12 slides:

1. `临床问题`
2. `现有证据`
3. `研究缺口`
4. `数据来源`
5. `方法概览`
6. `主要结果`
7. `验证与稳健性`
8. `解释性`
9. `与同类研究对比`
10. `局限性`
11. `投稿定位`
12. `需要老师决策的问题`

## Proposal / Defense

Use 12-15 slides:

1. 背景与意义
2. 国内外研究现状
3. 科学问题
4. 研究目标
5. 数据与对象
6. 方法路线
7. 预实验/已有结果
8. 可行性
9. 创新点
10. 风险与替代方案
11. 时间计划
12. 预期成果

## Existing Deck Cleanup

When editing old slides:

- Preserve the original storyline unless the user asks for restructuring.
- Remove prompt-like comments and private drafting notes.
- Move long explanation into speaker notes.
- Standardize terminology across slides.
- Add a `表述边界` slide if the project involves unpublished results, external validation, clinical claims, or causal inference.
